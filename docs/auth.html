Users Posts Auth

Base Route: /auth


JWT has been implemented for all routes with the exception of auth/login and the POST route of /users
. To login as a user send a POST request with a JSON request body that has the parameters username and password. Username would be your email. Password is plain text format. Do not send it hashed.

Passwords are stored hashed as BCrypt for security reasons. Do not send any passwords hashed, your requests will fail. Send as plain text when sending them.

When sending a request to the server(outside of the routes that require authentication) include this in your header with the actual token you've received when logging in.
Authorization : Bearer eyJhbGciOiJIUzI1NiJ9.eyJzdW-SHORTENED FOR CONTEXT - twukSlLw4X4kAOs9vnAvpE0

Role based authentication has been implemented. Basic, Management, and Admin. The default user role is Basic.


Auth Routes

All routes accept and return JSON

POST - "/auth/login Login as a user. Return a JWT
GET - "/auth/validate returns a response code 202 if the JWT is valid.
